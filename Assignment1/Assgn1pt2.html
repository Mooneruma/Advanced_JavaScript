Window Object Review


  • Window Object

    • The window object represents an open window in a browser. By calling the window object multiple methods can be used.


  • Navigator Object

    • The navigator object contains information about the browswer. Can be used to check setting of the browser for things such as whether or not cookies are enabled. Can also be called on to determine what type browser your page is being displayed on.


  • location Object

    • The location object contains information about the current URL being used. The object is apart of the window object and has to be accessed by using window.location property.


  • screen Object

    • This is used to access information about the users screen. Useful for setting the size of websites. Can be used to determine height and width of the users screen.


  • history Object

    • The history object contains the URLs visited by the user accessed using the window browser. This to must be used through the window property to access. User history is usually apart of the browser.


  • document Object

    • When a website is loaded into a web broswer the webpage it presents is the document object. Everything withen the document object then becomes nodes for the webpage to be produced. The Document object allows javascript access to all of its properties and methods to access the content from withen.


DOM Review

  • Document Object

    • The Dom is loaded from the html that is recieved into the browser. The Dom And CSSOM "css object" are on top of what is called a rendering tree. In otherwise the Dom and Css each have a tree for what order things are rendered. The Document Object in short is the base of your entire webpage.


  • How the browser renders

    • The Broswer renders by taking in the html the broswer recieves and then building the DOM. When it loads the DOM it also loads the css included. Once these begin rendering everything withen them then starts to render in order of thier rendering trees. As each part renders they are a part of the document object appering on your page.


  • Node

    • A node is technically what makes up most of the wepage including the document, every html element , the text, attributes, and all comments are different types of nodes. These nodes inherit thier properites from the parents or the top node which is called the root. Nodes can have a number of children in which they pass thier properties to.


  • Tree Structure

    • When a web page is loaded a DOM is created laying out a model for the website. When the model is loaded it creates a tree of objects. With this document object model Java Script has the ability to access the dom to to get all the power it needs to edit dynamic HTML. By using this tree you can also determine a pattern of inheritence.


  • event model or event handlers

    • Event model allows you to use javascipt to act on diffenet event handlers. Such as when an onclick event is triggered the event wont wend until the javascript has been executed. This can allow you to creat multiple methods to work with your javascipt functions such as mouse over, hover ,focus and many others.
